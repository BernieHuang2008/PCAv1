Personal Cryptographic Architecture (PCA)

Version: v1.0

设计理念

整个系统以一个永久离线的 Master 为唯一信任根，通过 KDF 构建可恢复的能力体系，通过 PKI 构建身份信任体系，并使用分级 Vault 保存所有无法派生的外部秘密。

Identity Branch

  • Identity Root 长期离线，仅用于签发年度身份。
  • 年度 Identity 用于签发各种用途身份（Email、Laptop、Phone 等）。
  • 邮件采用一次性身份（One-time Identity），默认不公开完整证书链，仅在需要时选择性披露。
  • 推荐算法：Ed25519（签名）；未来可升级到后量子算法。

Encrypt Branch

Generation

  • 所有可确定性恢复的秘密均由 HKDF 派生。
  • 包括钱包熵、密码管理器主密钥、文件加密主密钥等。

推荐算法

对象推荐
Master256-bit 随机种子
KDFHKDF-SHA-512
Identity 签名Ed25519
Vault 加密XChaCha20-Poly1305（推荐）
文件加密XChaCha20-Poly1305 或 AES-256-GCM
BitcoinBIP39 + BIP32

核心原则

  1. 唯一根密钥。
  2. 最小长期密钥。
  3. 身份、派生、存储三者职责分离。
  4. 风险分级（Alpha / Bravo / Charlie）。
  5. 默认最小公开，支持选择性证明。
